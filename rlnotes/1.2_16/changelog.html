This release can be deployed to the Google Application Engine (GAE), however there are certain restrictions to doing so.

  • The application must use a view technology other than JSP (i.e. Facelets)
  • The com.sun.faces.enableMultiThreadedStartup configuration option must be disabled (see the table below)

New context init parameters added for this release:

Paramter Name Description Default Value
com.sun.faces.enableAgressiveSessionDirtying If true, make it so on every request to JSF, every session scope attribute is touched. This has a severe performence impact, but it guarantees that session replication occurs, even in the case when a session key/value pair is not modified, but data that is referenced by that key/value pair is modified. false
